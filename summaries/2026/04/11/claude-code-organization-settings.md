## 概要

本資料は、Claude Code Meetup Tokyo（2026年4月）にてメルカリのAI Security TeamのHiroki Akamatsが発表した内容です。Claude Codeはファイル操作・Web検索・コマンド実行などの強力な機能を持つため、PCの認証情報や重要ファイルへの不正アクセスといったセキュリティリスクが生じます。メルカリでは組織全体へのMDM（端末管理）による設定配布を通じて、エンジニアと非エンジニアの異なるニーズに対応しながら、安全なClaude Code利用環境を整備した取り組みを紹介しています。

## 主要ポイント

- **Claude Codeのリスク**: ファイル操作・Web検索・コマンド実行機能により、PC上の認証情報（~/.aws, ~/.ssh等）や重要ファイルへのアクセスがセキュリティ上の脅威になり得る
- **セキュリティ設定の基本方針**: 人間による確認の必須化、危険なコマンド実行の制限、システム変更（sudo）の禁止、ディレクトリ・ネットワーク制限の4軸で対応
- **MDMによる組織配布**: エンジニアと非エンジニアで需要が異なるため、端末管理ツールを用いて設定を一斉配布
- **カスタマイズ可能な設定と安全な既定設定の分離**: ユーザーが変更可能な設定と、組織として強制する安全な既定設定を明確に分けて提供
- **AI Security Teamの役割**: 外部AIツールのセキュリティ対策、安全なAI利用ガイドライン策定、プラットフォーム整備を担当
- **エンジニア向け柔軟性の確保**: 開発業務への影響を最小限にしながら、セキュリティを確保する設定設計
- **組織的なAI利用ガバナンス**: 個人任せにせず、組織レベルでClaude Codeの安全な利用基盤を整備する重要性を提言
