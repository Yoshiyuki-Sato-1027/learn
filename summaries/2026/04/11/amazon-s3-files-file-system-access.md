## 概要

Amazon S3 Filesは、S3バケットをファイルシステムとして直接アクセスできるようにする新サービスです。Amazon EFSをベースに構築されており、S3の拡張性・耐久性・コスト効率を維持しながら、ファイルシステムセマンティクスを提供します。コードの変更なしに既存のファイルベースアプリケーションがS3データにアクセスでき、データの移行も不要です。2026年4月7日に34のAWSリージョンで一般提供が開始されました。

## 主要ポイント

- **ファイルシステムとしてのS3アクセス**: NFS 4.1/4.2プロトコルをサポートし、POSIXパーミッション・ファイルロック・read-after-write一貫性を提供
- **インテリジェントなルーティング**: 128KB以下の小ファイルはサブミリ秒〜数ミリ秒の低レイテンシストレージから提供、128KB超の大規模読み取りはS3から直接ストリーミング（最大数TB/秒のスループット）
- **双方向同期**: ファイルシステムとS3バケット間で変更が自動的に同期される
- **同時接続**: EC2、Lambda、ECS、EKS、Fargate、Batchなど最大25,000のAWSコンピュートリソースが同一ファイルシステムに同時接続可能
- **データ不要なマイグレーション**: 既存のS3バケットをそのまま使用でき、データはS3 APIとファイルシステム両方から同時アクセス可能
- **自動期限切れ**: 未使用データは高性能ストレージから自動削除（設定可能：1〜365日、デフォルト30日）
- **セキュリティ**: 転送中はTLS暗号化、保存時はAWS KMS暗号化（AWS管理キーまたはカスタマー管理キー）
- **主なユースケース**: AIエージェントのメモリ永続化、ML データ準備ワークロード、ファイルベースアプリケーションのS3直接利用、データ複製不要な共有アクセス
- **料金モデル**: 高性能ストレージ上のデータのみ課金。128KB以上の読み取りはS3 GETリクエスト料金のみ（ファイルシステム料金なし）
- **リージョン提供**: 34のAWSリージョンで一般提供中
